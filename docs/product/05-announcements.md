# Announcements

## What Announcements Are

Announcements are notices that admin sends to students. They appear on the student dashboard and on a dedicated announcements page. They are used for things like:

- Exam date reminders
- Platform updates and new features
- Course-specific news (e.g. "New questions added to General Nursing bank")
- Important alerts (e.g. "Planned maintenance this Saturday")
- Promotional messages (e.g. "50% off RN Premium this week")

## Targeting — Who Sees an Announcement

Admin can control exactly who sees each announcement using targeting options:

- **All students** — everyone on the platform sees it
- **By programme** — only students in a specific programme (e.g. only RN students, or only RM and RPHN)
- **By course** — only students who have access to a specific course
- **By level** — only students at a certain year level (e.g. Level 200 students only)
- **By subscription kind** — only PAID subscribers, or only students on TRIAL
- **By product** — only students on a specific product (e.g. "RN Premium 30 Days" subscribers)
- **By cohort** — only students from a specific intake year (e.g. 2024 cohort)
- **Specific users** — hand-picked individual students by name or ID

When multiple targeting options are set at the same time, the student must match **all** of them to see the announcement. For example, if an announcement targets "Programme: RN" and "Level: 200", only RN students who are also in Level 200 will see it. An RN student in Level 100 would not see it, and a Level 200 RM student would not see it either. This is AND logic — every condition must be met.

## Announcement States

Each student's interaction with an announcement is tracked:

- **Unread** — the student has not interacted with the announcement yet
- **Read** — the student clicked "Mark as Read"
- **Clicked** — the student clicked a link or call-to-action button in the announcement
- **Dismissed** — the student dismissed the announcement (hides it from their view)

These states are per-student. One student might have read an announcement while another has not seen it yet.

## Pinned Announcements

Admin can pin an announcement. Pinned announcements:

- Always appear at the top of the list, above non-pinned ones
- Stay visible and prominent until unpinned
- Are useful for urgent or time-sensitive notices that every student should see first

## Scheduling

Admin can set a publish date and an unpublish date on any announcement:

- The announcement only appears to students during that window
- Before the publish date — it exists in the system but is not visible to students
- After the unpublish date — it disappears from student views automatically
- This is useful for time-limited messages like "Exam registration closes Friday" that should not linger after the deadline passes

## Dashboard Strip

The student dashboard always shows a small strip of the most recent unread announcements:

- Up to 2 unread announcements are shown
- Pinned announcements appear first
- A "View all" link shows the total count of unread announcements and takes the student to the full announcements page
- This ensures students see important notices without having to actively check the announcements page
